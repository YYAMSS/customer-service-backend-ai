# 测试执行计划（流水线 · 待办勾选）

依据：`doc/需求对照测试文档.md`。  
用途：按阶段顺序编写/执行测试；**每一项为最小粒度（单步、不可再拆）**；完成一项将 `- [ ]` 改为 `- [x]`。  
中断恢复：找到**第一个未勾选**项，从该项继续。

**勾选约定**

- `- [ ]` 未完成  
- `- [x]` 已完成  

---

## Pipeline Stage 0 — 测试前置（数据、配置、客户端）

- [x] **PL-001** — 从 `edu-data` / 仓库样本中摘录并固定：至少 1 个有效课程名、1 个班次名、1 个有效订单号、1 个学员标识（写入测试常量或 fixture，保证可复现）。
- [x] **PL-002** — 在测试配置中写入被测服务 `BASE_URL`（及鉴权头/Token 占位，若项目需要）。
- [x] **PL-003** — 实现或引用「创建 HTTP 客户端」封装（超时、错误体解析），供后续用例共用。
- [x] **PL-004** — 实现或引用「非流式发送对话请求」单次调用封装（对应 TC-5.2-01）。
- [x] **PL-005** — 实现或引用「流式 SSE 消费」单次调用封装，并能拼出完整正文（对应 TC-5.2-02）。
